01

Smart HR Attendance System with Face Recognition

📋 Case Overview

This mobile attendance solution transforms how organizations track employee presence by replacing physical biometric terminals with intelligent smartphone-based check-in. The mobile application leverages the device's native Camera API to capture facial images, processes them through on-device machine learning models for identity verification, and uses Location Services (FusedLocationProviderClient) to ensure employees are physically present at designated work locations. The system demonstrates seamless integration of multiple mobile platform APIs: CameraX for consistent camera access across devices, TensorFlow Lite for real-time face detection and recognition (processing locally to protect biometric privacy), and Firebase Firestore for cloud data synchronization. Event-driven architecture handles camera lifecycle through OnClickListener triggers, processes location updates via LocationCallback, and manages Firebase operations with OnSuccessListener/OnFailureListener patterns. The UI employs Material Design components including EditText for employee ID input, MaterialButton with loading states, RecyclerView with DiffUtil for efficient attendance history display, and CardView elements for dashboard summaries. Layout implementation uses ConstraintLayout for the main screen's flat hierarchy, FrameLayout to overlay UI controls on the camera preview, and CoordinatorLayout for smooth scrolling behaviors with collapsing toolbar effects.

⚙️ Technical Architecture & Mobile Platform APIs

📷 CameraX API - Camera Hardware Access
Mobile API Demonstration: CameraX provides lifecycle-aware camera operations that automatically handle camera opening, closing, and configuration based on Activity/Fragment lifecycle. The ProcessCameraProvider binds camera use cases (Preview + ImageAnalysis) to the lifecycle owner.

Implementation: PreviewView displays live camera feed. ImageAnalysis.Analyzer receives camera frames as ImageProxy objects at 30fps, converts YUV_420_888 format to Bitmap, then feeds to ML model. setTargetRotation() ensures correct image orientation across device rotations. The camera initializes asynchronously, avoiding ANR (Application Not Responding) errors on the main thread.
📱 FusedLocationProviderClient - Location Sensor API
Mobile API Demonstration: This Google Play Services API abstracts GPS, Wi-Fi, and cellular location providers into a unified interface. It automatically selects the best location source based on accuracy requirements and battery constraints.

Implementation: LocationRequest configured with setPriority(PRIORITY_HIGH_ACCURACY) for GPS precision and setInterval(10000) for updates every 10 seconds. LocationCallback receives Location objects containing latitude, longitude, and accuracy. The app uses Location.distanceTo() to calculate distance from office coordinates, implementing geofence logic that compares computed distance against 100-meter radius threshold before enabling check-in button.
🤖 TensorFlow Lite - On-Device ML API
Mobile API Demonstration: TensorFlow Lite enables on-device machine learning inference without cloud dependency, maintaining privacy and working offline. Models are loaded from assets and run using the Interpreter class with optional GPU acceleration.

Implementation: Interpreter loads .tflite models via MappedByteBuffer from assets folder. GpuDelegate passed to Interpreter.Options enables hardware acceleration for 10x faster inference. Input preprocessing converts Bitmap to ByteBuffer/FloatBuffer matching model's input tensor shape. Interpreter.run(inputBuffer, outputBuffer) executes inference. For face recognition, output contains face embeddings compared against enrolled user embeddings using cosine similarity. Confidence threshold of 0.85 prevents false acceptances.
☁️ Firebase Firestore - Cloud Database API
Mobile API Demonstration: Firestore provides real-time NoSQL cloud database with automatic synchronization and offline persistence. Changes propagate instantly to all connected clients via WebSocket connections.

Implementation: FirebaseFirestore.getInstance() provides database reference. CollectionReference and DocumentReference navigate data hierarchy. addDocument() creates new records, setDocument() overwrites, updateDocument() modifies fields. addSnapshotListener() enables real-time updates with DocumentSnapshot callbacks. Offline persistence enabled via FirebaseFirestoreSettings allows full CRUD operations without connectivity. Data structure: users/{userId}/attendance/{date} contains timestamp, coordinates, and confidence score fields. Transaction API ensures atomic multi-document updates.

📱 UI Components Used

  • EditText – inputType="number" for employee ID, TextInputLayout wrapper shows floating label and error messages via setError()
  • MaterialButton – setEnabled(false) during processing, ProgressBar inside shows loading state, OnClickListener triggers check-in workflow
  • TextView – Updates via setText() in callback handlers, uses Spannable for colored status messages (green/red)
  • RecyclerView – LinearLayoutManager for vertical list, DiffUtil calculates minimal updates, ViewHolder pattern with data binding
  • CardView – Material elevation (8dp), rounded corners (cardCornerRadius), contains CheckIn/Out times and location accuracy
  • ProgressBar – Indeterminate circular style, visibility toggled VISIBLE/GONE during face recognition processing

🎨 Layout Architecture

  • ConstraintLayout – Root layout with horizontal/vertical chains for button groups, Guidelines at 50% for centered alignment, barriers for dynamic content
  • FrameLayout – Stacks PreviewView (camera) with overlay ImageView (face detection rectangle drawn using Canvas)
  • LinearLayout – android:orientation="vertical" with layout_weight for proportional space distribution in login form
  • CoordinatorLayout – Enables CollapsingToolbarLayout with scroll flags (scroll|exitUntilCollapsed), FAB uses anchor for positioning

⚡ Event Handling

  • View.OnClickListener – button.setOnClickListener { } lambda captures button click, checks permissions, initiates camera + location requests
  • LocationCallback – onLocationResult(LocationResult) receives location updates, extracts lastLocation, compares coordinates using haversine formula
  • ImageAnalysis.Analyzer – analyze(ImageProxy) called per frame, image.planes[0] contains Y channel, converts to Bitmap, runs TFLite inference
  • OnSuccessListener – Firebase Task.addOnSuccessListener callback receives DocumentReference, updates UI on main thread via runOnUiThread
  • OnFailureListener – addOnFailureListener receives Exception, checks if FirebaseNetworkException for offline handling, shows Snackbar error

🔑 Key Technical Insights

  • Using CameraX instead of deprecated Camera API reduces code complexity by 60% and eliminates device-specific bugs
  • On-device ML processing with TensorFlow Lite keeps face data private and works in areas with poor connectivity
  • FusedLocationProviderClient's smart location strategy extends battery life by 40% compared to raw GPS
  • Firebase's offline persistence ensures zero data loss even during network outages
  • Background WorkManager syncs pending attendance records automatically when connectivity returns

Vehicle Wash IoT Automation System

📋 Case Overview

This self-service carwash automation system demonstrates advanced mobile connectivity APIs by creating a mobile control interface for industrial IoT hardware. The application uses Bluetooth Low Energy (BLE) API to establish wireless communication with wash station PLCs, sending GATT characteristic writes to trigger equipment (water jets, soap dispensers, rotating brushes). ML Kit Barcode Scanning API processes vehicle QR codes through Camera API integration, extracting JSON-encoded wash specifications. The app's network layer implements MQTT protocol via Eclipse Paho mobile client for real-time bidirectional communication with cloud servers. UI architecture employs BottomNavigationView for primary navigation between scanner, active sessions, and history screens. The scanner interface uses FrameLayout to overlay scan guidelines on the CameraSourcePreview, while active wash sessions display in RecyclerView with custom ViewHolders showing real-time progress. Event handling includes BluetoothGattCallback for BLE connection state changes and characteristic notifications, BroadcastReceiver monitoring network connectivity via ConnectivityManager, and MqttCallback handling message delivery confirmations. Layout strategy uses ConstraintLayout with Guidelines for responsive positioning across 7-10 inch industrial tablets, and MotionLayout for animated transitions between scanning and wash-in-progress states with interpolated progress bars and status text updates.

⚙️ Technical Architecture & Mobile Platform APIs

📡 Bluetooth Low Energy (BLE) API - Hardware Communication
Mobile API Demonstration: BLE stack uses BluetoothAdapter.getDefaultAdapter() to access device Bluetooth radio. BluetoothLeScanner performs device discovery with ScanCallback receiving ScanResult objects containing RSSI signal strength and advertised service UUIDs.

Implementation: BluetoothGatt.connectGatt() establishes connection to PLC controller. BluetoothGattCallback.onConnectionStateChange() detects STATE_CONNECTED, then discoverServices() enumerates available GATT services. BluetoothGattCharacteristic.setValue(bytes) prepares command payload, writeCharacteristic() transmits to hardware. onCharacteristicWrite() callback confirms transmission success. Characteristic notifications (setCharacteristicNotification) enable real-time status updates from equipment, with onCharacteristicChanged() receiving sensor data like water pressure and soap levels.
📷 ML Kit Barcode Scanning API
Mobile API Demonstration: ML Kit Vision API provides fast on-device barcode/QR code detection without requiring separate model downloads. Supports 1D barcodes, QR codes, Data Matrix, and more with automatic format detection.

Implementation: InputImage.fromBitmap() or fromMediaImage() creates image input from camera frames. BarcodeScanner.process(inputImage) asynchronously detects codes, returning Task>. Barcode objects contain getRawValue() for decoded string, getBoundingBox() for screen coordinates, and getValueType() identifying format (QR_CODE, EAN_13, etc.). Processing typically completes in 50-100ms. Success handler extracts vehicle specifications from JSON-encoded QR data.
🌐 ConnectivityManager - Network Monitoring API
Mobile API Demonstration: ConnectivityManager system service monitors network state changes, allowing apps to respond to connectivity transitions between Wi-Fi, cellular, and offline states.

Implementation: NetworkCallback registered via registerNetworkCallback() receives onAvailable(), onLost(), and onCapabilitiesChanged() events. NetworkCapabilities.hasCapability(NET_CAPABILITY_INTERNET) verifies actual internet connectivity beyond network connection. When offline detected, app queues MQTT publish operations in Room database. NetworkRequest.Builder specifies required capabilities (TRANSPORT_WIFI, TRANSPORT_CELLULAR). On connectivity restore, DataSyncWorker triggered via WorkManager uploads pending transactions.
📨 Eclipse Paho MQTT Client
Mobile API Demonstration: MQTT (Message Queuing Telemetry Transport) is a lightweight publish-subscribe protocol ideal for IoT applications. Eclipse Paho provides mobile-compatible MQTT client implementation.

Implementation: MqttAndroidClient initialized with broker URL (tcp://broker.hivemq.com:1883) and unique client ID. MqttConnectOptions configures credentials, keep-alive interval, and clean session flag. connect() establishes connection with MqttCallback providing connectionLost(), messageArrived(), and deliveryComplete() callbacks. publish() sends messages with QoS levels: 0 (at most once), 1 (at least once), 2 (exactly once). subscribe() listens to topics using wildcards: carwash/station1/# receives all subtopics. Retained messages ensure new subscribers receive last known state.

📱 UI Components Used

  • MaterialToolbar – inflateMenu(R.menu.main) loads menu items, onMenuItemClick listener handles settings/logout actions, setNavigationIcon shows back button
  • FloatingActionButton – app:srcCompat for vector drawable icon, setOnClickListener launches QR scanner, animates with show()/hide() based on scroll
  • TextView – Data binding with @{viewModel.bleStatus} updates text automatically, SpannableString for mixed styling (bold device name)
  • Snackbar – make(view, message, LENGTH_LONG).setAction("Retry") for dismissible alerts, anchorView positions above FAB
  • BottomNavigationView – menu resource defines items, setOnItemSelectedListener switches fragments via FragmentTransaction.replace()
  • Chip – Closeable style with closeIcon, ChipGroup holds multiple chips, onCloseIconClick disconnects BLE device

🎨 Layout Architecture

  • ConstraintLayout – app:layout_constraintTop_toBottomOf chains views vertically, bias (0.3) shifts elements, aspect ratio constraint (16:9) for preview
  • FrameLayout – Match_parent dimensions allow fullscreen scanner, child views with layout_gravity="center" overlay scan guidelines
  • MotionLayout – MotionScene XML defines transitions with KeyFrameSet, KeyAttribute animates alpha/scale, onTransitionCompleted listener detects state changes

⚡ Event Handling

  • BluetoothGattCallback – onConnectionStateChange(gatt, status, newState) detects STATE_CONNECTED/DISCONNECTED, onServicesDiscovered enumerates characteristics, onCharacteristicWrite confirms command transmission
  • View.OnClickListener – Manual control buttons write specific byte arrays to BLE characteristics: byteArrayOf(0x01) starts pump, 0x00 stops
  • BroadcastReceiver – IntentFilter(ConnectivityManager.CONNECTIVITY_ACTION) registered in onResume, onReceive checks NetworkInfo.isConnected()
  • MqttCallback – messageArrived(topic, MqttMessage) parses JSON payload, deliveryComplete(IMqttDeliveryToken) confirms QoS 1 message sent
  • BarcodeDetectorProcessor – Barcode.getBoundingBox() provides Rect, getRawValue() returns decoded string, Vibrator.vibrate(100) gives haptic feedback

🔑 Key Technical Insights

  • BLE's GATT protocol allows controlling multiple wash modules simultaneously through different characteristics
  • MQTT's lightweight overhead (2-byte headers) makes it perfect for high-frequency status updates without data bloat
  • Local Room database caching prevents revenue loss when network connectivity drops during transactions
  • ML Kit's automatic QR detection works 3x faster than legacy ZXing library and handles damaged codes better
  • Implementing exponential backoff for BLE reconnections prevents battery drain from aggressive retry loops

Field Workforce Management Application

📋 Case Overview

This field workforce management application showcases mobile offline-first architecture capabilities and background processing APIs for enterprise mobile solutions. The system leverages Room Persistence Library (SQLite abstraction) with @Entity, @Dao, and @Database annotations to store job assignments, customer data, and service history locally, enabling full functionality without network connectivity. Google Maps SDK integrates via SupportMapFragment, displaying technician locations with custom marker icons and polyline route history. Location tracking uses FusedLocationProviderClient with LocationRequest configured for PRIORITY_BALANCED_POWER_ACCURACY, updating every 120 seconds via foreground Service with ongoing notification to comply with platform background location restrictions. Geofencing API registers CircularGeofence objects with 200-meter radius around job sites, triggering PendingIntent broadcasts when technicians enter/exit zones (GEOFENCE_TRANSITION_ENTER/EXIT). WorkManager schedules PeriodicWorkRequest for data synchronization running every 15 minutes under NetworkType.CONNECTED constraint. UI implements NavigationDrawer with DrawerLayout and NavigationView for menu navigation. Main screen uses TabLayout + ViewPager2 with FragmentStateAdapter managing Today/Upcoming/Completed job fragments. RecyclerView displays jobs with ItemTouchHelper enabling swipe gestures—swiping right calls customer via Intent.ACTION_DIAL, left launches navigation via Intent.ACTION_VIEW with geo URI. Event handling includes OnMapReadyCallback for map initialization, OnMarkerClickListener opening job details in BottomSheetDialogFragment, and LiveData observers triggering UI updates when Room database changes.

⚙️ Technical Architecture & Mobile Platform APIs

🗺️ Google Maps SDK - Location Visualization
Mobile API Demonstration: Maps SDK integrates via SupportMapFragment added to layout XML or programmatically. getMapAsync() callback provides GoogleMap instance after asynchronous initialization, avoiding main thread blocking.

Implementation: GoogleMap.addMarker() creates Marker objects with custom BitmapDescriptor icons (colored by technician status). MarkerOptions.position() sets LatLng coordinates. Polyline objects visualize routes with PolylineOptions.addAll(List). OnMarkerClickListener intercepts marker taps, returning false to show default InfoWindow or true for custom handling. CameraUpdateFactory.newLatLngBounds() animates map viewport to fit all markers. MapStyleOptions.loadRawResourceStyle() applies JSON styling customizing road colors, label visibility, and POI display. Local tile caching enables partial offline map viewing of previously loaded areas.
📍 Fused Location Provider + Geofencing API
Mobile API Demonstration: FusedLocationProviderClient intelligently combines GPS, Wi-Fi, and cell tower data for optimal battery-efficient location tracking. Geofencing API provides event-driven location monitoring without continuous polling.

Implementation: LocationRequest created with setInterval(120000) for updates every 2 minutes, setPriority(PRIORITY_BALANCED_POWER_ACCURACY) balances accuracy (~100m) with battery life. requestLocationUpdates() with LocationCallback receives onLocationResult() events. CircularGeofence built with latitude, longitude, radius (200 meters), and transition types (GEOFENCE_TRANSITION_ENTER | EXIT). GeofencingClient.addGeofences() registers geofences with PendingIntent triggering BroadcastReceiver when technician enters/exits job sites. This eliminates need for continuous GPS polling, reducing battery consumption by 70%.
💾 Room Persistence Library
Mobile API Demonstration: Room provides type-safe SQLite database abstraction with compile-time query verification and seamless LiveData integration for reactive UI updates. Essential for offline-first mobile applications.

Implementation: @Entity annotation defines database tables with primary keys and indexes. @Dao interfaces declare queries: @Query("SELECT * FROM jobs WHERE date = :date"), @Insert, @Update, @Delete methods. @Database abstract class extends RoomDatabase, defines version number and entity list. Room.databaseBuilder() creates database instance. Foreign keys enforce referential integrity. @Embedded flattens nested objects. @TypeConverter handles complex types. LiveData> queries automatically notify observers when data changes. Migration strategies handle schema updates between versions without data loss.
⚙️ WorkManager - Background Task API
Mobile API Demonstration: WorkManager provides guaranteed background task execution with constraint-based scheduling, surviving app process death and device reboots. Compatible across all platform versions using JobScheduler, AlarmManager, or BroadcastReceiver as appropriate.

Implementation: PeriodicWorkRequest.Builder(DataSyncWorker::class.java, 15, TimeUnit.MINUTES) creates recurring work. Constraints.Builder().setRequiredNetworkType(CONNECTED).setRequiresBatteryNotLow(true) defines execution requirements. enqueueUniquePeriodicWork() prevents duplicate scheduling. Worker's doWork() runs on background thread, returning Result.success(), Result.retry(), or Result.failure(). OneTimeWorkRequest for immediate execution when critical updates needed. WorkInfo.State observable via WorkManager.getWorkInfoByIdLiveData() for UI progress updates. Worker queries Room for syncStatus="PENDING" records, uploads via Retrofit REST client, updates local database on successful network response.

📱 UI Components Used

  • NavigationView – app:headerLayout displays user profile, app:menu inflates navigation items, setNavigationItemSelectedListener handles menu selections
  • RecyclerView – ListAdapter with AsyncListDiffer for background diffing, ItemTouchHelper.SimpleCallback(0, LEFT | RIGHT) enables swipe actions, notifyItemChanged animates updates
  • FloatingActionButton – app:layout_anchor="@id/coordinator" positions relative to parent, setBehavior hides on scroll, extended FAB shows text + icon
  • SupportMapFragment – getMapAsync(OnMapReadyCallback) provides GoogleMap instance, MarkerClusterRenderer groups nearby markers, UiSettings.setZoomControlsEnabled(true)
  • TabLayout – TabLayoutMediator.attach() links to ViewPager2, tab.setText() sets titles, addOnTabSelectedListener handles manual tab switches
  • BottomSheetDialogFragment – Peeks at 40% height, drag to expand, shows job details with customer info, parts list, and "Start Job" button

🎨 Layout Architecture

  • DrawerLayout – Root layout with child NavigationView (layout_gravity="start"), openDrawer()/closeDrawer() programmatic control, setScrimColor() darkens content
  • CoordinatorLayout – Behavior classes coordinate animations: AppBarLayout.ScrollingViewBehavior links RecyclerView scroll to toolbar collapse, FAB uses app:layout_dodgeInsetEdges
  • ConstraintLayout – Inside job detail screens, Guideline at 30% creates left column for labels, Flow helper arranges action buttons horizontally with wrapping
  • CardView – RecyclerView item layout, app:cardElevation animated from 2dp to 8dp on press, app:cardUseCompatPadding ensures consistent spacing

⚡ Event Handling

  • OnMapReadyCallback – onMapReady(GoogleMap) called asynchronously after map loads, executes googleMap.setMapType(MAP_TYPE_NORMAL), adds markers in batch with addAll()
  • GoogleMap.OnMarkerClickListener – onMarkerClick(Marker) receives clicked marker, marker.getTag() retrieves associated Job object, returns true to prevent default info window
  • ItemTouchHelper.Callback – onSwiped(ViewHolder, direction) checks if LEFT (navigate) or RIGHT (call), creates Intent with ACTION_DIAL/ACTION_VIEW, startActivity()
  • LocationCallback – Override onLocationResult(), iterate LocationResult.locations, last element is most recent, post to LiveData for observer notification
  • BroadcastReceiver (Geofence) – GeofencingEvent.fromIntent(intent) extracts event, getGeofenceTransition() returns ENTER/EXIT, getGeofencingRequest() identifies job site
  • LiveData> Observer – observe(lifecycleOwner) { jobs -> adapter.submitList(jobs) } automatically updates RecyclerView when Room database changes

🔑 Key Technical Insights

  • Geofencing API reduces battery consumption by 70% compared to constant GPS polling for arrival detection
  • Room's @Transaction annotation ensures customer + job updates succeed or fail atomically, preventing data corruption
  • WorkManager's retry with exponential backoff handles poor connectivity areas without manual intervention
  • Using PRIORITY_BALANCED_POWER_ACCURACY instead of HIGH_ACCURACY extends daily battery life from 6 hours to full shift
  • Clustering map markers (when 10+ jobs in area) improves map performance and reduces visual clutter dramatically

AI-Powered Photo & Video Editing Application

📋 Case Overview

This AI-powered photo enhancement application (inspired by Remini's viral success) demonstrates sophisticated mobile Camera APIs, hardware-accelerated media processing, and on-device machine learning integration. The app uses Camera2 API for low-level camera control, creating CameraCaptureSession with multiple output surfaces for simultaneous preview and high-resolution capture. TensorFlow Lite Interpreter with GpuDelegate processes images through neural networks for enhancement tasks: super-resolution upscaling, face restoration, old photo colorization, and scratch removal. OpenGL ES shaders handle real-time filter preview by rendering to GLSurfaceView with custom fragment shaders applying color matrices, convolution kernels for sharpening/blur, and tone curve adjustments. MediaStore API manages photo gallery access and saving enhanced images with proper EXIF metadata preservation. UI implements ViewPager2 with TabLayoutMediator for horizontal swiping between Edit/Enhance/Filters tabs. The editing screen uses ConstraintLayout with horizontal RecyclerView for filter thumbnails below main ImageView preview. SeekBar components provide real-time adjustment of enhancement intensity with OnSeekBarChangeListener triggering shader uniform updates. Event handling includes GestureDetector for pinch-to-zoom on ImageView using ScaleGestureDetector.OnScaleGestureListener, double-tap for before/after comparison via OnDoubleTapListener, and TextureView.SurfaceTextureListener managing OpenGL rendering surface lifecycle. Layout architecture employs MotionLayout for animated transitions between camera and editing modes with keyframe-based property animations for smooth UI element repositioning.

⚙️ Technical Architecture & Mobile Platform APIs

📷 Camera2 API - Advanced Camera Control
Mobile API Demonstration: Camera2 uses pipeline architecture with CameraManager.openCamera() providing CameraDevice handle. CameraCharacteristics queries hardware capabilities (supported resolutions, focus modes, ISO range). Multiple ImageReader surfaces enable concurrent preview + full-resolution capture.

Implementation: CameraCaptureSession.setRepeatingRequest() streams preview frames. CaptureRequest.Builder configures per-frame parameters: CONTROL_AE_MODE controls auto-exposure, CONTROL_AF_MODE sets autofocus behavior, LENS_FOCUS_DISTANCE for manual focus. ImageReader.OnImageAvailableListener receives Image objects containing YUV_420_888 pixel data in Image.Plane arrays. Image.getCropRect() and getTimestamp() provide metadata. Converting YUV to RGB: extract Y, U, V planes, apply color space transformation matrix. For enhancement processing, RGB bitmap feeds TensorFlow Lite model Input Tensor, output tensor contains enhanced pixel values reconstructed as Bitmap.
🎥 MediaRecorder & MediaCodec APIs
Mobile API Demonstration: MediaRecorder provides simple video recording API for standard capture scenarios. MediaCodec exposes low-level hardware encoders/decoders, enabling frame-by-frame processing for AI filter application. Essential for applying effects during recording or post-processing.

Implementation: MediaRecorder with setVideoEncoder(HEVC) and setVideoSize(3840, 2160) captures 4K at 30fps, using H.265 codec for 50% smaller files than H.264. For AI effects pipeline: MediaExtractor.readSampleData() extracts encoded frames, MediaCodec.createDecoderByType("video/avc") decodes to Surface, AI model processes in GPU via TensorFlow Lite, MediaCodec.createEncoderByType("video/hevc").createInputSurface() receives processed frames, queueInputBuffer() with presentation timestamps maintains sync. Configure with KEY_COLOR_FORMAT = COLOR_FormatSurface for zero-copy GPU-to-encoder transfer. Asynchronous callbacks (onInputBufferAvailable, onOutputBufferAvailable) handle pipeline flow control.
🤖 TensorFlow Lite - On-Device Machine Learning API
Mobile API Demonstration: TensorFlow Lite provides mobile ML inference without cloud dependency. Interpreter loads .tflite model files from assets folder using FileInputStream mapped to MappedByteBuffer. GPU acceleration via GpuDelegate constructor passed to Interpreter.Options.

Implementation: Bitmap resized to model's input dimensions (typically 256x256 or 512x512) using Bitmap.createScaledBitmap(). Pixel values extracted into FloatBuffer or ByteBuffer depending on model quantization. Interpreter.run(inputBuffer, outputBuffer) executes inference. For photo enhancement, input is original image, output is enhanced pixel array. INT8 quantized models use ByteBuffer input with values 0-255, Interpreter converts internally. Multi-threaded inference with setNumThreads(4) improves CPU performance. NNAPI delegate accelerates inference on dedicated neural processing units when available. Model execution measures with SystemClock.elapsedRealtime() for performance monitoring.
🎮 OpenGL ES 3.0 - GPU Graphics API
Mobile API Demonstration: OpenGL ES provides direct GPU access for hardware-accelerated rendering, enabling real-time image effects impossible on CPU. Fragment shaders execute in parallel across GPU cores, processing millions of pixels simultaneously. Essential for 60fps video filter preview.

Implementation: GLSurfaceView creates OpenGL context. GLSurfaceView.Renderer implements onSurfaceCreated() (compiles GLSL shaders with glCompileShader(), links program with glLinkProgram()), onSurfaceChanged() (sets viewport), and onDrawFrame() (executes per-frame rendering). Fragment shaders written in GLSL version 300 es apply pixel operations: uniform sampler2D for input texture, texture(sampler, texCoord) samples pixels, vec4 output is processed color. Two-pass Gaussian blur: first pass renders horizontally blurred image to framebuffer object (FBO), second pass reads FBO as texture and blurs vertically. glUniform1f() passes parameters (blur radius, filter intensity) from Java to shader. glTexImage2D() with GL_TEXTURE_EXTERNAL_OES and EXT_YUV_target extension allows direct YUV texture binding from camera without RGB conversion, saving 15ms per frame.

📱 UI Components Used

  • TextureView – Camera preview with getSurfaceTexture(), SurfaceTextureListener.onSurfaceTextureAvailable() signals ready for camera, supports Matrix transformations for zoom/pan
  • ImageView – setImageBitmap() displays processed images, setScaleType(FIT_CENTER) maintains aspect ratio, Matrix.setScale() for pinch-zoom with GestureDetector
  • RecyclerView – Horizontal LinearLayoutManager (HORIZONTAL), SnapHelper centers selected filter, ViewHolder loads filter thumbnails with Glide, onClick applies filter to preview
  • VideoView – MediaController for play/pause/seek, setVideoURI() loads from content:// URI, OnPreparedListener detects ready state, OnCompletionListener handles playback end
  • SeekBar – setMax(100) for 0-100% intensity, onProgressChanged(seekBar, progress, fromUser) checks fromUser=true, calculates alpha: progress/100f, updates shader uniform
  • Slider (Material) – addOnChangeListener { slider, value, fromUser } provides float values, setValueFrom(-2f) to setValueTo(2f) for exposure compensation in EV stops

🎨 Layout Architecture

  • ConstraintLayout – app:layout_constraintDimensionRatio="16:9" maintains camera preview aspect, Barrier prevents control buttons from overlapping preview, ConstraintSet programmatically animates layout changes
  • ViewPager2 – android:orientation="horizontal", setOffscreenPageLimit(1) keeps adjacent fragments in memory, TabLayoutMediator synchronizes tabs with pages
  • MotionLayout – MotionScene defines with constraintSetStart/End, defines motion path, transitionToState(R.id.editing) triggers animation
  • FrameLayout – Holds GLSurfaceView for OpenGL rendering overlaid with control buttons using android:layout_gravity to position corners, foreground drawable shows darkened scrim

⚡ Event Handling

  • GestureDetector – onDoubleTap() toggles before/after comparison, onSingleTapConfirmed() shows/hides UI controls, onFling(velocityX, velocityY) detects swipe direction and speed
  • ScaleGestureDetector – onScale(detector) returns detector.scaleFactor, multiply current zoom by scaleFactor, clamp between 1.0f (no zoom) and maxZoom (typically 10.0f), apply to ImageView Matrix
  • View.OnClickListener – Capture button onClick checks if camera ready, calls takePicture(), disables button, shows progress indicator, re-enables after onImageSaved callback
  • SurfaceTextureListener – onSurfaceTextureAvailable(surface, width, height) receives Surface, use as CameraDevice.createCaptureSession() target, onSurfaceTextureSizeChanged handles rotation
  • SeekBar.OnSeekBarChangeListener – onProgressChanged updates filterIntensity variable, if (fromUser) triggers immediate reprocessing, calling applyFilter() which runs TFLite model with new alpha blending ratio
  • GLSurfaceView.Renderer – onSurfaceCreated() compiles GLSL shaders, onDrawFrame() called per frame, binds textures, sets uniform variables (filter intensity, LUT texture), draws fullscreen quad

🔑 Key Technical Insights

  • GPU Delegate accelerates TensorFlow Lite models 8-12x faster than CPU inference, enabling real-time AI effects
  • Using YUV_420_888 format throughout pipeline eliminates expensive color space conversions, saving 40ms per frame
  • OpenGL ES shader compilation happens on first launch; caching compiled shaders reduces subsequent cold starts by 75%
  • MediaCodec's Surface input mode uses zero-copy rendering, processing 4K video with 3GB RAM instead of 8GB
  • Quantized INT8 TensorFlow models maintain 98% accuracy while being 4x smaller and 3x faster than FP32 models
  • Two-pass Gaussian blur in fragment shaders achieves 60fps on 1080p video, impossible with CPU-based approaches
